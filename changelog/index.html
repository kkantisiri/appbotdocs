Changelogs

Updates

AppBot is always getting updated, to make it easier for you to know what's getting updated, the change logs are listed here.

Version 1.6.7 - /mydata minor bug fix

  • Fixed a bug whre applications without an autorole set would create a bug

Version 1.6.6 - /mydata

  • Added /mydata for server's applications

Version 1.6.5 - Caps Bug Fix

  • Fixed a bug where caps while editing an application would make it uneditable

Version 1.6.4 - Application Name Editing

  • Application names can now be edited

Version 1.6.3 - Logs Bug

  • Fixed a bug where accepted applications wouldn't get logged or archieved

Version 1.6.2 - /register enhanced

  • If you're using the bot as a register bot, /reister will now make more sence. Instead of something like "Apply for Position", it'll be changed to "Register for position" based off of the command invoked.

Version 1.6.1 - Bug fix

  • Fixed a minor bug where trying to whitelist a role before blacklisting would cause a bug to occur

Version 1.6.0 - Major bug fixes and minor changes and updates

  • Fixed a major lasting bug where timing out while creating an application would create a permanently bugged application
  • Fixed a major lasting bug where you can review an application while the applier was still creating the application
  • Added /help <command> to view a specific command's description
  • Made /format and /apps in the same channel rather than in a DM
  • Changed "amount" to "num_of_qns" in the usage for /app create(Less confusing)
  • Added /register alias for /apply

Version 1.5.5 - Minor bug fixes

  • Un-unupdated an update where the cool down message should've sent
  • Another minor change

Version 1.5.4 - Major Security Bug Fix

  • Fixed a bug where members without permissions can use /app subcommands

Version 1.5.3 - Archived Applications

  • Added /setarchieves <channel> - set a channel where a copy of applications get sent to the end, after an app gets reviewed
  • Fixed the message in command cooldown - The bot was supposed to send a message if you ran a command while it was on cooldown

Version 1.5.2

Closed applications will now be crossed out like this in /app list and /apply
