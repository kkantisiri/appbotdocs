Changelogs

Updates

AppBot is always getting updated, to make it easier for you to know what's getting updated, the change logs are listed here.

Version 1.6.7 - /mydata minor bug fix

  • Fixed a bug whre applications without an autorole set would create a bug

Version 1.6.6 - /mydata

  • Added /mydata for server's applications

Version 1.6.5 - Caps Bug Fix

  • Fixed a bug where caps while editing an application would make it uneditable

Version 1.6.4 - Application Name Editing

  • Application names can now be edited

Version 1.6.3 - Logs Bug

  • Fixed a bug where accepted applications wouldn't get logged or archieved

Version 1.6.2 - /register enhanced

  • If you're using the bot as a register bot, /reister will now make more sence. Instead of something like "Apply for Position", it'll be changed to "Register for position" based off of the command invoked.

Version 1.6.1 - Bug fix

  • Fixed a minor bug where trying to whitelist a role before blacklisting would cause a bug to occur

Version 1.6.0 - Major bug fixes and minor changes and updates

  • Fixed a major lasting bug where timing out while creating an application would create a permanently bugged application
  • Fixed a major lasting bug where you can review an application while the applier was still creating the application
  • Added /help <command> to view a specific command's description
  • Made /format and /apps in the same channel rather than in a DM
  • Changed "amount" to "num_of_qns" in the usage for /app create(Less confusing)
  • Added /register alias for /apply

Version 1.5.5 - Minor bug fixes

  • Un-unupdated an update where the cool down message should've sent
  • Another minor change

Version 1.5.4 - Major Security Bug Fix

  • Fixed a bug where members without permissions can use /app subcommands

Version 1.5.3 - Archived Applications

  • Added /setarchieves <channel> - set a channel where a copy of applications get sent to the end, after an app gets reviewed
  • Fixed the message in command cooldown - The bot was supposed to send a message if you ran a command while it was on cooldown

Version 1.5.2

  • Closed applications will now be crossed out like this in /app list and /apply

Version 1.5.1

  • Unreverted an application - accidentally removed discrims from /apps, added back

Version 1.5.0 - Major Update

  • Added a /edit command for member answer editing
  • Fixed some text issues
  • Fixed a bug where non-utf-8 answers that were longer than 1024 chars would cause a bug
  • Added a more descriptive log message for new applications
    Before: Member appplied in this server.
    After: Member applied for Job
  • Added /blacklist to set a role that can be given to members to prevent them from applying
  • Added /delete to allow members to delete apps that they've submitted
  • Added /app open <app> & /app close <app> for temporarily closing app positions
  • "Enter a reason that you denied the app" is now embedded
  • Denying app reason timeout has been extended to 10 minutes
  • Global announcements can now be configurable
  • Fixed a small text error where the prefix would not be the correctly invoked one in

Version 1.4.6

  • Fixed a weird bug that would happen if someone tried to edit an app without an acceptance message

Version 1.4.5

  • Fixed a bug where missing members caused /apps to stop working - now they don't show up anymore

Version 1.4.4

  • Added descriminators in /apps to make finding users easier

Version 1.4.3

  • Added a rotating status, fixed the /info command

Version 1.4.2

  • Fixed a pretty common bug where appliers enter a negative number if the server has multiple job positions

Version 1.4.1

  • Added a gif for autorole permissions help

Version 1.4.0

  • Added an error handler for people who disabled DMs

Version 1.3.4

  • Fixed the status of the bot

Version 1.3.3

  • All commands are no longer case sensitive

Version 1.3.2 - Minor Updates to Help Command

  • Added /support command for an invite link to the support server

Version 1.3.1

  • Added /uptime

Version 1.3 - Updated /help command

  • The help command is more modern, organized, categorized, and detailed.

Version 1.2.3

  • Fixed a minor bug where editing application's custom accept messages before 1.2.1 would cause an error

Version 1.2.2 - Bug fixes and Design Improvements

  • A major bug that prevented editing questions within /config

Version 1.2.0 - Considerably Major Update

  • Handler made for image uploading
  • Theming (/settheme hex) will change the colors of all embeds for the bot
  • Numbering added to application editing
  • Made as presence status for the bot
  • Fixed a bug where deleting an application while members still have applications for the deleting position would cause an error

Version 1.1.3

  • Added Embed Limits handler
  • Logs are now in Beta

Version 1.1.2

  • The time to reply with an answer has been extended to 30 minutes

Version 1.1.1 - Minor Update/Bug Fixes

  • Fixed a bug where servers that didn't appear on the database would start spamming the console, fixed by creating a handler that adds them to the database as if they just added the bot
  • Updated the new /help command in according to update 1.1.0

Version 1.1.0 - Set a role for Application Reviewing

  • You can now set a role to be able to edit and view applications with /setrole @Role

Version 0.9.2 - Minor Update

  • Automated Bug Log has been added for faster bug fixes

Version 0.9.1 - New Update

  • Custom emotes to make reactions look fancier
  • Autoroles if a member gets accepted for a role
  • Fixed /help command for members
  • Other minor fixes

Version 0.9.0 - Multiapplications

  • Multi-applications are now supported! You can now have more than one application format for your members! To add another application, type /config

Version 0.8.3 - Application Formats

  • Added /format, so that appliers can easily view the application to get an idea of what questions that they will be answering

Version 0.8.2

  • Added a feature where appliers can now look over what they put in the application in the end, and able to edit any mistakes they make

Version 0.8.1

  • The file that holds the data has been shortened all the way up to from originally almost 40000 lines to around 105

Version 0.8.0 - Initial Public Release

  • After first releasing the bot, the bot has gotten added to a huge server :)
